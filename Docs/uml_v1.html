Item<T> Template

Generic item with one type-parameterized special-data field

  • m_weight : float
  • m_data : T

  • GetName() const : const std::string&
  • GetDescription() const : const std::string&
  • GetValue() const : int
  • GetWeight() const : float
  • GetData() const : const T&
  • Display() const : std::string
  • No default constructor — items must be fully specified
  • All template definitions live in Item.h

Instantiations (locked)

  • Item<int> — Medkit (heal amount in HP)

Container<T, MaxSize> Template

Bounded inventory container | non-type N for compile-time cap

Members

  • m_items : std::vector<T>
  • MaxSize is template parameter — not stored as a member

Methods

  • Container() = default
  • Add(const T& item) : bool
  • Remove(size_t index) : bool
  • Find<Pred>(p) : iterator
  • Find<Pred>(p) const : const_iterator
  • Sort<Comp>(c) : void
  • GetCount() const : size_t
  • IsFull() const : bool
  • IsEmpty() const : bool
  • begin() / end() (×2: const + non-const)

Notes

  • Wraps std::vector — cap is policy enforced in Add(), not by vector itself
  • Find wraps std::find_if | Sort wraps std::sort
  • Predicate / comparator templated for inlining (vs std::function)

Relationships

  • Container<T,N> ◆→ holds many T internally (composition via std::vector)
  • InventoryManager ◆→ holds many Container instances (composition via std::map) — exact storage type TBD
  • Game ◆→ owns m_playerBag (a Container), m_stash (an InventoryManager), and m_loot (vector of LootOnGround)
  • Renderer → reads const Game& at draw time. Game has no reference to Renderer. One-way dependency.
  • No inheritance anywhere. If you find yourself adding it during the build, stop and ask why.
